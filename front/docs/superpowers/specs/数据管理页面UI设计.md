# 数据管理页面 UI 设计

- 状态：已确认，待实现
- 日期：2026-07-08
- 范围：仅前端 UI 设计（数据继续使用示例 / ionosphere），不含后端 API 与真实上传/导出/删除逻辑
- 目标文件：`src/DatasetManagementPage.tsx`、`src/styles.css`、`src/App.tsx`（路由）、`src/dashboard/navigation.ts`（如需）

## 1. 背景与目标

当前「数据管理」页面（`DatasetManagementPage`）是**单数据集流程式**布局：状态条 → 4 个概览卡 → 左(导入区)+右(统计表+基础聚类选择)两栏，全部使用本地示例数据。其视觉形态与分析工作台（summary-grid / metrics-grid / top-insight-grid / secondary-card-grid 多列卡片堆叠）高度相似，两者都呈"卡片堆叠"观感，难以一眼区分用途。

本次重设计将数据管理页面改为**多数据集目录**模型，并刻意与工作台做视觉差异化：以行式表格目录 + 两栏详情页取代卡片堆叠。

## 2. 设计决策（已与用户确认）

1. **核心模型**：多数据集目录。列表为主体，点行进入该数据集的全页详情页。
2. **实现范围**：仅前端 UI。数据继续使用示例（ionosphere 及若干示例数据集），后端 API 留作后续。
3. **视觉关系**：刻意与工作台差异化。
4. **列表列**：名称/时间、规模(n/m/c)、状态/类型、使用情况（四组全选）。
5. **详情入口**：点行进入全屏详情视图，通过组件内 `selectedId` 状态切换（不新增路由），返回切回目录。
6. **初始状态**：工具栏 + 表格常态（不做大型上传引导空状态）。

## 3. 视觉差异化策略

| 维度 | 工作台（现状） | 数据管理（本设计） |
| --- | --- | --- |
| 主形态 | 多列卡片网格 | 单列密集行式表格 |
| 边界 | 圆角卡片 + 玻璃边框 | 无卡片框、靠分隔线 |
| 密度 | 低（大 padding / gap） | 高（行高紧凑） |
| 装饰 | blur-ball 背景、3D 矩阵 | 极简、近乎扁平 |
| 字号 | 标题大、卡片正文 13–14px | 表格 12–13px、数字等宽 |

目录页几乎不使用 `.panel` 卡片（仅工具栏是一个浅条）；详情页才引入少量轻卡片，且采用**左固定窄栏 + 右长流内容**的两栏布局，而非上下堆叠——从根上与"卡片堆叠"区分。

## 4. 目录页布局（`/workbench/datasets`）

### 4.1 结构

```
┌─────────────────────────────────────────────────────────────┐
│ 工具栏  [搜索数据集…]  [类型:全部▾]  [示例数据]  [上传数据集] │
├──┬──────────┬───────────┬──────────┬──────────────┬───────┤
│  │ 数据集   │ 规模 n/m/c│ 状态/类型│ 使用情况     │       │
│  ├──────────┼───────────┼──────────┼──────────────┤       │
│  │ ionosphere│351/351/2  │●有标签 数值│ 3 任务 · 07-08│  ›  │
│  │ wine      │178/178/3  │●有标签 数值│ 1 任务 · 06-21│  ›  │
└──┴──────────┴───────────┴──────────┴──────────────┴───────┘
```

### 4.2 工具栏 `.dataset-toolbar`

- 一条浅色横条容器（非 `.panel`，无圆角卡片框），与表格连成一体。
- 左：搜索框（占主宽，placeholder「搜索数据集…」）。
- 中：类型下拉筛选（全部 / 数值 / 混合）。
- 右：`使用示例数据`次按钮（`.btn.btn-secondary`）、`上传数据集`主按钮（`.btn.btn-primary`）。
- 不加顶部状态条（旧单数据集流程的 `.status-header` 残留不再出现）。

### 4.3 表格 `.dataset-table`

- 无外层卡片框，表头与行之间、行与行之间靠分隔线区分（`border-bottom: 1px solid var(--app-border)`）。
- 列定义（从左到右）：
  1. **数据集名称**：主行名称 + 副行小字创建时间。
  2. **规模**：`n / m / c` 三段等宽数字（`font-variant-numeric: tabular-nums`），如 `351 / 351 / 2`。
  3. **状态/类型**：状态徽章（有真实标签=绿点+「有标签」；无=灰点+「无标签」）+ 类型文字（数值 / 混合）。
  4. **使用情况**：`N 任务 · 最后分析 日期`；无使用则灰字「未使用」。
  5. **进入提示**：`›` 箭头，右对齐。
- 字号：表头 12px、行 13px、数字等宽。
- 交互：
  - 行 hover 整行底色高亮（`var(--primary-8)`）。
  - 点行 → `navigate('/workbench/datasets/:id')`。
  - 空数据时表格中央一行提示：「暂无数据集，点击右上角上传」。

### 4.4 目录页明确不做的事

- 不使用 `.summary-grid` / `.metric-card` / `.panel` 等工作台卡片组件。
- 不做大型上传引导空状态（按用户决策：工具栏 + 表格常态）。

## 5. 详情页布局

详情页不引入新路由，而是在 `DatasetManagementPage` 组件内部用本地状态 `selectedId` 切换：`selectedId === null` 显示目录视图，否则显示该数据集的详情视图；点行 `setSelectedId(id)`，详情页「返回」`setSelectedId(null)`。

> 实现决策（已确认）：采用页面内状态切换而非嵌套路由。原因：当前工作台架构为 `/workbench/:section` 单段路由 + `activeSection` 字符串条件渲染（见 App.tsx:1296、1433），无嵌套路由；页面内切换改动最小、不破坏侧栏高亮（`getActiveWorkbenchSection` 精确匹配），代价是详情页无独立 URL（刷新回目录）——纯前端示例阶段可接受。

### 5.1 结构（两栏，非堆叠）

```
┌─────────────────────────────────────────────────────────────┐
│ ← 返回   数据集 · ionosphere                                 │
├──────────────┬──────────────────────────────────────────────┤
│ 左栏(320px)  │ 右栏(1fr)                                     │
│ [元信息卡]   │ ─ 标签分布 ─────────────────────────────────  │
│ [规模数字卡] │ ─ 基础聚类统计 ─────────────────────────────  │
│ [操作按钮组] │ ─ 选择基础聚类 ─────────────────────────────  │
└──────────────┴──────────────────────────────────────────────┘
```

### 5.2 顶栏

- `← 返回`（`setSelectedId(null)`，回到目录视图）+ 面包屑「数据集 · {name}」。

### 5.3 左栏 `.dataset-detail-sidebar`（320px 固定）

纵向三个**轻卡片**（详情页仅有的卡片，窄列）：

1. **元信息卡**：名称、上传时间、类型、状态徽章。
2. **规模数字卡**：`351 样本 / 351 基础聚类 / 2 真实类别` 三行大数字。
3. **操作按钮组**：
   - `创建任务`（主按钮，基于此数据集进入任务创建——本次仅 UI，跳转/回调留 TODO）
   - `导出 数据`（次按钮，TODO）
   - `重命名`（次按钮，TODO）
   - `删除`（`.btn.btn-danger` + 确认——本次仅 UI，留 TODO）

### 5.4 右栏 `.dataset-detail-main`（1fr）

三个**分段区块** `.dataset-detail-section`（非卡片，靠标题 + 分隔线区分，避免堆叠观感）：

1. **标签分布**：类别条形图（每行：类别名 + 横条 + 计数）。复用示例 `labelDistribution`。
2. **基础聚类统计**：表格，复用现有 `.dataset-stat-table` 样式（列：基础聚类 / 簇数量 / 标签范围 / 选择状态）。复用示例 `clusterStats`。
3. **选择基础聚类**：复用 `.selection-controls` / `.selection-meter` / `.selected-base-grid` 的样式语言与交互（n_base 输入、随机种子、随机选择按钮、进度条、已选网格）。

### 5.5 与工作台区分

工作台是横向并列多卡片；详情页是**左固定窄栏 + 右长流内容**，纵向阅读为主。

## 7. 数据与示例集

为支撑"多数据集目录"，前端示例数据由当前单个 ionosphere 扩充为若干条示例数据集（保持本地、无 API）。每条至少含：id、名称、创建时间、n/m/c、是否有真实标签、类型、任务数、最后分析时间、标签分布、clusterStats。

具体示例集可在实现阶段补全，本设计只约束字段结构与最小数量（≥3 条以体现目录列表）。

## 8. 验收标准

1. 访问 `/workbench/datasets` 看到工具栏 + 行式表格目录，≥3 条示例数据集，四组列齐全。
2. 点任一行在页面内切换到详情视图（URL 不变，`selectedId` 置为该 id），显示返回 + 面包屑 + 两栏布局（左 320px 元信息/规模/操作，右 标签分布/统计表/选择聚类）；点返回切回目录。
3. 目录页表格行 hover 整行高亮；详情页分段区块靠分隔线区分，无卡片堆叠观感。
4. 与分析工作台并排对比，二者主形态（行式表格 vs 卡片网格）一眼可区分。
5. 按钮可点但动作留 TODO（创建任务/导出/重命名/删除），不报错、不假成功。
6. `npm run build` 与 `npx tsc --noEmit` 通过，现有测试不回归。
